Linked SLA Alerts

Linked SLA Alerts notifies linked Jira issues when this issue has an SLA, so other queues stay in the loop without needing access to the host project. From the SLA relay panel you send this issue’s SLA status and expiration to linked tickets—each gets a Jira comment with status (At-Risk, Breached, or In SLA range), date of expiration, and @mentions for assignees. Optional Slack sends the same summary with clickable links to the posted tickets.

Atlassian Forge App

Key Features

  • SLA relay panel — Notify linked tickets of this issue’s SLA status and expiry. Shows how many issues are linked; choose “All linked tickets” or “Only these” (specific keys), then click “Send SLA to linked tickets.”
  • Send to linked tickets — For each chosen linked ticket, the app posts one Jira comment with status (At-Risk, Breached, In SLA range), date of expiration, and @mentions assignee (and reporter if different).
  • Send to all or specific tickets — Send to all linked tickets or list which issue keys to notify (e.g. OCV2-4, OCV2-5).
  • Optional Slack — Same message to a Slack channel with clickable links to the posted tickets. When Slack is enabled in admin, one Slack message includes “(Posted to: [links to tickets]).”
  • At-risk/breach alerts — When a linked issue’s SLA goes at risk or breached, the app can post on the parent (current) issue and notify assignee, reporter, watchers, and send to Slack/email (configurable).
  • Admin configuration — Triggers, who to notify, Jira comments, Slack (webhook or Bot token), custom templates, and optional SLA field ID when auto-detect doesn’t find your SLA field. Jira Service Management SLA is supported (API fallback).

How It Works

You’re on the host ticket (the issue that has the SLA). The SLA relay panel shows how many issues are linked. If this issue has no SLA data, you’ll see: “This issue has no SLA data. Add an SLA to this issue to send it to linked tickets.” Once there is SLA data, you choose whether to send to all linked tickets or only specific keys, then click “Send SLA to linked tickets.” Each selected linked ticket receives one comment with the status wording (At-Risk, Breached, In SLA range), the date of expiration (or “—” when not available), and @mentions. No table or cross-queue visibility is required—you relay this issue’s SLA to the tickets that need to know.

Why It Helps

Support, escalation, and engineering queues often depend on work in other queues. Linked SLA Alerts lets the queue that owns the SLA keep other queues informed without granting them access to the host project. Best for teams that need to relay SLA status and due dates to linked work across queues or projects.
