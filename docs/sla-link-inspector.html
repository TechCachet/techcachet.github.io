Linked SLA Alerts

Version 1.1.15 | Atlassian Forge App for Jira

Linked SLA Alerts notifies linked tickets when a parent issue’s SLA goes at risk or breached. The app lives on the parent issue that owns the SLA. Linked tickets do not need their own SLA.

Getting Started

  1. Install Linked SLA Alerts from the Atlassian Marketplace.
  2. Open any Jira issue that has an SLA and linked tickets.
  3. The Linked SLA Alerts panel appears in the issue sidebar.
  4. Click Configure app in the panel to set up triggers and notifications.

How It Works

The app operates on the parent issue (the ticket with the SLA). When that issue’s SLA hits a configured trigger, the app posts a comment on each linked ticket and @mentions the configured people on that linked ticket.

Linked tickets receive notifications about the parent’s SLA. They do not need their own SLA field.

Configuration

Go to Jira admin settings and select Linked SLA Alerts under Marketplace apps, or click Configure app in the panel.

SLA Field (Optional)

The app auto-detects SLA fields by name. If your site uses a non-standard field name or a different language, enter the custom field ID (for example, customfield_10005) to ensure the app reads the correct SLA.

Trigger Conditions

Configure when alerts fire on linked tickets:

  • At risk — fires when the parent issue’s SLA enters at-risk status
  • Breached — fires when the parent issue’s SLA is breached
  • 30 minutes remaining — fires when 30 minutes remain on the parent’s SLA (uses the At Risk notify list)
  • Only notify if linked issue is still open — skips alerts for resolved or done linked tickets

Who Gets Notified

Configure separately for At Risk and Breached triggers. Notifications are posted as comments on each linked ticket and @mention:

  • Assignee
  • Reporter
  • Watchers
  • Additional users (search by name or email)
  • Users from custom fields (for example, Request Participants)

Notification Channels

  • Jira Comment — posts a comment on each linked ticket
  • Jira @Mention — includes @mentions in the comment
  • Slack — send via incoming webhook URL or Bot token and channel ID
  • Email webhook — send via Zapier, Make, or similar webhook URL

Message Templates

Two templates are configurable:

Alerts template — used when automatic triggers fire. Available variables:

  • {{issueKey}} — the parent issue key
  • {{slaName}} — SLA field name
  • {{remainingTime}} — time remaining or overdue
  • {{status}} — at risk, breached, or 30 minutes remaining
  • {{assignee}} — @mentions as configured

Send SLA Alert to Linked Tickets template — used when you click the button in the panel. Available variables:

  • {{issueKey}} — the parent issue key
  • {{slaStatus}} — At-Risk, Breached, or In SLA range
  • {{expiryDate}} — formatted expiration date
  • {{assignee}} — @mentions of the linked ticket’s assignee and reporter

Send SLA Alert to Linked Tickets Panel

From any parent issue with linked tickets, the panel lets you send an on-demand SLA status update:

  • Choose All linked tickets or select Only these and enter specific issue keys.
  • Click Send SLA to linked tickets.
  • The app posts a comment on each selected linked ticket with the parent’s SLA status and expiry date.

Security

Linked SLA Alerts runs on Atlassian Forge. Core processing and app configuration use Atlassian infrastructure; Tech Cache does not operate separate long-lived app servers for that core logic.

  • Configuration and operational state are stored in Forge Key-Value Store (encrypted at rest per Atlassian).
  • When you enable Slack or webhook/email integrations, payloads are sent only to endpoints you configure—subject to those providers’ policies.
  • Slack bot tokens are masked in API responses and never sent to the browser.
  • Tokens can be cleared at any time via Clear saved token in the Configurations page.

Slack Setup

Option 1 — Incoming Webhook:

Create a webhook at api.slack.com, then paste the URL into the Slack webhook field in Configurations.

Option 2 — Bot Token and Channel:

  • Create a Slack app at api.slack.com.
  • Under OAuth and Permissions, copy the Bot User OAuth Token.
  • Paste the token into the bot token field in Configurations.
  • Enter the channel ID (open the channel in Slack, click the channel name, and select View channel details).

FAQ

Do linked tickets need their own SLA?
No. The app uses only the parent issue’s SLA. Linked tickets just need to exist as standard Jira issue links on the parent.

Does the app work across projects?
Yes. Linked tickets can be in any project on the same Jira site.

What happens if a linked ticket is already resolved?
If “Only notify if linked issue is still open” is enabled in Configurations, the app skips resolved or done tickets.

Where is my Slack token stored?
In Atlassian Forge Key-Value Store, encrypted at rest by Atlassian. It is never sent to the browser or stored outside Atlassian infrastructure.

How do I find my SLA custom field ID?
In Jira, go to Project settings, then Fields, and copy the field ID for your SLA field.
